🌳 分类算法

判断西瓜好坏

4.1 决策树 🌳

像医生问诊一样

📖 算法原理：

就像医生看病一样，通过问一系列问题来诊断！

  • 从根节点开始：问第一个最重要的问题（如"根蒂蜷缩吗？"）
  • 根据答案分支：是 → 走左边，否 → 走右边
  • 继续提问：每个节点继续问下一个问题
  • 到达叶节点：得出最终结论（好瓜✅或坏瓜❌）

💡 决策过程：白色方框是判断节点（提问），绿色方框✅是"好瓜"结论，红色方框❌是"坏瓜"结论。从上往下，根据每个问题的答案走不同的分支，最终到达结论！

根蒂蜷缩吗？
↙ 是 否 ↘
敲声浊响吗？
坏瓜 ❌
↙ 是 否 ↘
好瓜 ✅
色泽乌黑吗？
↙ 是 否 ↘
好瓜 ✅
坏瓜 ❌

4.2 K近邻算法 (KNN) 👥

近朱者赤，近墨者黑

📖 算法原理：

想判断一个西瓜是好是坏？看看它周围K个"邻居"是什么类型就行了！

  • 第1步：找到离待判断西瓜最近的K个西瓜（比如K=3）
  • 第2步：数一数这K个邻居中，好瓜和坏瓜各有多少
  • 第3步：少数服从多数，哪种多就判断为哪种

💡 动画说明：图中黄色星星🍉是待判断的西瓜，绿色✅是好瓜，红色❌是坏瓜。点击按钮后，会画出K=3邻域范围，找到最近的3个邻居来投票决定！

4.3 支持向量机 (SVM) ✂️

画一条最佳分界线

📖 算法原理：

想象在两堆不同的瓜之间画一条线，把它们分开！

  • 目标：找到一条线，把好瓜和坏瓜完美分开
  • 要求：这条线要离两边的瓜都尽可能远（最大间隔）
  • 支持向量⭐：离分界线最近的关键点（橙色菱形）

💡 图示说明：紫色虚线是决策边界（分界线），绿色✅是好瓜区域，红色❌是坏瓜区域。两条浅色虚线是间隔边界，橙色⭐标记的是支持向量（决定边界位置的关键点）。

4.4 朴素贝叶斯 (Naive Bayes) 🎲

用概率来推理决策

📖 算法原理：

根据已知特征，计算它是好瓜或坏瓜的概率，选择概率大的那个！

  • 第1步：统计历史数据中好瓜和坏瓜的比例（先验概率）
  • 第2步：计算在好瓜中，某特征出现的概率（条件概率）
  • 第3步：同样计算在坏瓜中，该特征出现的概率
  • 第4步：用贝叶斯公式综合计算，比较哪个概率更高

💡 "朴素"的含义：假设各个特征之间相互独立（比如颜色和形状互不影响），这个假设简化了计算，但在很多实际情况下效果依然很好！

第六章

📊 模型评估

考试打分

🎯 混淆矩阵

实际：好瓜 实际：坏瓜
预测：好瓜 真正例 TP: 40 假正例 FP: 5
预测：坏瓜 假反例 FN: 3 真反例 TN: 52

📏 评估指标

🎯 准确率 (Accuracy)

准确率 = 92%

准确率 = (TP + TN) / (TP + TN + FP + FN)

= (40 + 52) / (40 + 52 + 5 + 3) = 92 / 100 = 92%

✅ 含义

预测正确的样本占总样本的比例

"考试得了多少分"

⚠️ 局限性

在数据不平衡时会误导

例如：100个样本，95个是A类
模型全预测为A，准确率95%
但对B类完全无效！

🤔 生活例子：
医生诊断100个病人，92个诊断正确 → 准确率92%
看起来不错，但如果漏诊了癌症患者呢？
所以还需要看精确率和召回率！

⚖️ 精确率 vs 召回率

精确率 (Precision)

88.9%

预测为好瓜的，有多少真是好瓜

"你说话有多靠谱"

召回率 (Recall)

93.0%

实际的好瓜，你找出了多少

"你有多灵敏"

F1分数 = 90.9%

精确率和召回率的调和平均数

📈 ROC曲线与AUC

AUC (曲线下面积):
0.9-1.0: 优秀 ⭐⭐⭐⭐⭐

第七章

⚠️ 进阶话题

过拟合 vs 欠拟合

📊 欠拟合 - 模型太简单

❌ 欠拟合问题

• 模型太简单，连训练数据都拟合不好

• 训练误差大，测试误差也大

• 解决方法：增加模型复杂度

📊 恰到好处 - 完美平衡

✅ 理想状态

• 既能拟合训练数据，又能泛化到新数据

• 训练误差小，测试误差也小

• 目标：找到这个平衡点！

📊 过拟合 - 模型太复杂

⚠️ 过拟合问题

• 模型太复杂，记住了训练数据的噪声

• 训练误差很小，但测试误差大

• 解决方法：正则化、Dropout、更多数据

🆚 深度学习 vs 传统机器学习

对比维度 传统机器学习 深度学习
特征工程 需要人工设计特征 自动学习特征 ✓
数据量需求 小数据也能工作 ✓ 需要大量数据
计算资源 普通电脑即可 ✓ 需要GPU加速
训练时间 快速（分钟级）✓ 较慢（小时到天）

适用场景 结构化数据 图像、语音、文本 ✓
